Descripción general del sistema

Nuestro equipo propone la creación de una página web donde las mujeres víctimas de abuso y/o acoso sexual pueden denunciar los acontecimientos que vivieron de manera anónima. El objetivo es proveer una plataforma segura en donde se dé valor a la voz de las víctimas y se cree una comunidad de apoyo. La principal función de nuestro sistema será publicar las historias que se compartan; sin embargo, hemos pensado que otros usuarios pueden acceder al sitio web a buscar apoyo e información sobre lo que vivieron, de tal manera que compartiremos artículos escritos por expertos en el tema y compartiremos el contacto de asociaciones y expertos que puedan proveer apoyo especializado. Personas que no hayan sido víctimas también pueden acceder a la plataforma para informarse sobre los casos que se viven en el país. Estos usuarios pueden no escribir notas, pero pueden leer los casos que existen y qué tipo de conductas evitar, así como cómo identificar a las víctimas y cómo brindar apoyo cuando sea necesario. Planeamos que nuestra plataforma abarque todos los ámbitos posibles, desde víctimas hasta no víctimas, así como mostrar a los escépticos que el problema es real y afecta a la mayoría de las mujeres en el país.

Problemática por resolver:

Abuso sexual Tipo de actividad o contacto sexual en el que no das tu consentimiento.

Las mujeres no cuentan con una plataforma definida para compartir sus experiencias. Se hace uso de las redes sociales para contar las vivencias que han tenido; sin embargo, al hacer esto exponen su imagen y se vuelven víctimas de comentarios incrédulos, misóginos, machistas y bullying cibernético. El segundo foco es crear un espacio de concientización hacia las mujeres proveyendo un sentido de apoyo al tener recursos que logran generar empatía y comunidad. Esta concientización se podría dar a traves de impactar de diferentes formas como articulos, estadísticas, representaciones visuales (mapa, graficas), y tambien a traves de concentrar en atención a las mujeres pos-denuncia.

Forma como piensan solucionar la problemática:

Nuestra página web permite a las mujeres compartir sus experiencias a manera de desahogo y prevención (para que otras mujeres no pasen por lo mismo) de manera anónima. Con lo anterior, eliminan la posibilidad de ser víctimas de ciberbullying y ataques de otros usuarios. De igual manera, al compartir su vivencia con la comunidad pueden sanar heridas generadas por el suceso vivido. Planeamos incluir filtros y moderadores que eviten los comentarios que ataquen o denigren a las colaboradoras.

Descripción de soluciones similares:

  1. #SayHisName (liga): Esta página web esuna expresión artística que permite a las víctimas de abuso sexual compartir sus experiencias para ayudar en el proceso de recuperación emocional. Se puede navegar a través de algunas historias y publicar tus propias experiencias.
  2. Redes sociales (Twitter e Instagram): A través del uso de hashtags, como #MeToo, las víctimas de abuso sexual han podido compartir sus historias y alcanzar personas en todo el mundo.
  3. Brave Miss World (liga): Esta página web, relacionada con la película Brave Miss World, funciona de la misma manera que #SayHisName. La víctimas de abuso pueden compartir sus historias y unirse al movimiento #IAmBrave para promover la página y apoyar a otras víctimas, a diferencia de #SayHisName, aquí las historias pueden recibir retroalimentación de parte de otros usuarios.La creadora de la página menciona que su objetivo es "Incrementar la concientización y crear un cambio significativo en la percepción de violaciones a través del mundo." Diferenciador:

El diferenciador de nuestra aplicación se basa en dos partes.

  1. Proveer de apoyo para la mejora psicológica y emocional de las personas que comparten un caso. Esto mediante artículos y contactos.
  2. Utilizar la información recopilada para generar insights sobre la situación sobre el acoso en México. Por ejemplo, un mapa de denuncia en el que, al verificar una cuenta, se pueda dar información sobre el caso y persona que realizó el acoso indicandolo en el mapa. Esta misma se podría actualizar en vivo para mostrar la frecuencia de las denuncias para lograr mayor impacto.

De esta manera, el objetivo es cumplir todas las partes del problema apoyando no solamente una vez que ya pasó el acoso, sino, poder prevenir de mejor manera y así mismo aportar un espacio para el uso de forma eficiente de la información.

Fragmento de código (Ejemplo)

Es importante tomar en cuenta que dado el MEAN stack utilizado en la clase, el proyecto consistiría en varios servicios y módulos. El siguiente código solo demuestra un ejemplo de app.js para nuestro backend. 


const express = require('express');
const cors = require('cors');
const path = require('path');
const logger = require('morgan');
const cookieParser = require('cookie-parser');
const bodyParser = require('body-parser');
const mongoose = require('mongoose');
// Assume configs are set.
const config = require('config');

const app = express();

// Create necessary server.
const server = require('http').createServer(app);

// Cors middleware.
app.use(cors());

// Only log in dev mode.
if (process.env.NODE_ENV === "dev") {
  app.use(morgan('dev'));
}

app.use(bodyParser.json());
app.use(bodyParser.urlencoded({ extended: false }));
app.use(cookieParser());

// Get port for the server.
const port = process.env.PORT || 3000;
server.listen(port, () => {
  console.log(`Server is listening on port: ${port}`);
});
